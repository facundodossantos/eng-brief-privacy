Privacy Policy for EngBrief

Last updated: February 24, 2026

Data Collection

EngBrief does not collect, store, or transmit any personal information. We do not use cookies, analytics, or any form of user tracking.

Data Storage

The extension stores the following data locally on your device using Chrome's local storage API:

  • RSS feed articles (titles, URLs, descriptions, publication dates)
  • Your filter preferences (selected sources and tags)
  • Your favorite articles

This data never leaves your device and is not accessible to us or any third party.

Network Requests

EngBrief makes network requests exclusively to the following public RSS feeds:

  • Slack Engineering (slack.engineering)
  • Netflix TechBlog (netflixtechblog.com)
  • Spotify Engineering (engineering.atspotify.com)
  • Engineering at Meta (engineering.fb.com)
  • AWS Architecture Blog (aws.amazon.com)
  • Uber Engineering (uber.com)
  • GitHub Engineering (github.blog)

These requests are solely to fetch publicly available RSS feed content. No user data is sent in these requests.

Permissions

EngBrief requires the following Chrome permissions:

  • storage — to save your preferences and favorite articles locally
  • alarms — to schedule automatic feed updates every 60 minutes

Third-Party Services

EngBrief does not use any third-party analytics, tracking, advertising, or data processing services.

Changes to This Policy

If we make changes to this privacy policy, we will update the "Last updated" date above.

Contact

For questions about this privacy policy, please open an issue on our GitHub repository.
